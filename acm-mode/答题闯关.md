# 答题闯关

[题目链接](https://kamacoder.com/problempage.php?pid=1480)

---
## 题目描述
某答题闯关节目有 `m` 道编号 `1~m` 的题目，需将所有题目分为两个非空轮次。`n` 名选手的答题范围是连续区间 `[Lᵢ, Rᵢ]`，  
只要选手能答对某一轮的所有题目（即该轮题目完全被其区间覆盖）即可晋级。求所有可能的划分方式中，最多能让多少名选手晋级。  
- 输入：第一行 `n` 和 `m`，接下来 `n` 行每行 `Lᵢ`和 `Rᵢ`  
- 输出：最多晋级人数  

>数据范围：`1≤n≤5×10⁴，1≤m≤1×10⁵`  


## 核心思路分析
1. **最关键的核心引理**  
> 对于任意题目集合 `S`，能覆盖 `S` 的选手数量 = 能覆盖区间 `[min(S), max(S)]` 的选手数量。  

**证明：**  
- 若选手覆盖 `S`，则其区间必然包含 `S` 的最小值和最大值，因此覆盖整个`[min(S), max(S)]`  
- 若选手覆盖`[min(S), max(S)]`，则 `S` 中所有题目都在该区间内，必然覆盖 `S`  

通俗理解：**集合 `S` 的晋级能力只由它的左右端点决定，中间有没有空隙完全不影响。这是整个问题的突破口，将看似无限的划分可能性压缩到了有限的区间形式。**  


2. **最优划分的两种唯一可能**  

>根据引理，任何划分都可以等价为 "**一个连续区间 + 它的补集**" 的形式，且晋级人数不减少。而这种形式只有两种本质不同的最优情况：  

**策略一：单点分割（最优情况 1）**  
- 将某一道题 `x` 单独作为一轮，剩余题目作为另一轮。  
- 能晋级的选手：所有区间包含 `x` 的选手（因为覆盖单点 `x` 的选手必然能通过第一轮晋级）  
- 另一轮的晋级选手只能是覆盖全部题目的选手，已经包含在上述数量中  
- 目标：找到被最多选手区间覆盖的单点 `x`  

**策略二：前缀后缀分割（最优情况 2）**    
- 将题目分为前缀`[1, k]`和后缀`[k+1, m]（1≤k<m）`。  
- 能覆盖前缀`[1, k]`的选手：必须满足`Lᵢ=1且Rᵢ≥k`（左端点必须是 `1`，右端点至少到 `k`）  
- 能覆盖后缀`[k+1, m]`的选手：必须满足`Rᵢ=m且Lᵢ≤k+1`（右端点必须是 `m`，左端点最多到 `k+1`）  
- 重复修正：同时覆盖全部题目的选手（`L=1且R=m`）会被两边各算一次，需要减去重复计数  
- 目标：枚举所有 `k`，找到最大的晋级人数之和  


3. **为什么没有其他更优情况？**  

- 若划分成中间区间 + 补集：中间区间的晋级人数必然≤其内部任意单点的覆盖人数（覆盖区间的选手一定覆盖单点，反之不成立），不如策略一  
- 若划分成不连续集合：根据引理，等价于其最小包围区间的划分，晋级人数不会增加   


## 算法详细步骤
**步骤 1：边界处理**  
- 若 m=1，无法划分为两个非空轮次，直接输出 0。  

**步骤 2：差分数组统计单点覆盖次数（策略一）**  
- 初始化差分数组`diff`，大小为 `m+2`（避免 `R+1` 越界）  
- 对每个选手区间 `[L, R]`，执行`diff[L]++`和`diff[R+1]--`  
- 计算前缀和得到每个点的覆盖次数，记录最大值  

**步骤 3：预处理前后缀覆盖数组（策略二）**  
- `count_L1[r]`：左端点为 `1` 且右端点为 `r` 的选手数量  
- `pref[k]`：左端点为 `1` 且右端点≥`k` 的选手数量（从后往前前缀和计算）  
- `count_Rm[l]`：右端点为 `m` 且左端点为 `l` 的选手数量  
- `suff[x]`：右端点为 `m` 且左端点≤`x` 的选手数量（从前往后前缀和计算）  
- `covered_all`：覆盖全部题目的选手数量（`L=1且R=m`） 

**步骤 4：枚举分割点计算策略二最优解**  
- 对每个分割点 `k（1≤k<m）`，计算当前晋级人数：  
- `current = pref[k] + suff[k+1] - covered_all`
- 记录最大值。  

**步骤 5：最终答案**
- 取策略一和策略二的最大值。


## 复杂度分析
**时间复杂度：`O(n + m)`**  
- 遍历 `n` 个选手更新数组：`O (n)`  
- 计算差分数组前缀和：`O (m)`  
- 预处理前后缀数组：`O (m)`  
- 枚举分割点：`O (m)`  

**空间复杂度：`O(m)`**  
- 使用了 `4` 个大小为 `m+2` 的数组  

**该算法在题目给定的数据规模下，C++ 运行时间约为 0.1 秒，远低于 1 秒的时间限制。**  


## 代码实现:

### ***C++***

```CPP
#include <iostream>
#include <vector>
using namespace std;

int main() {
    //输入输出优化
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;
    if (m == 1) {
        cout << 0 << '\n';
        return 0;
    }

    //差分数组
    vector<int> diff(m + 2, 0);

    //count_L1[i]:左边界为1，右边界为i的区间数量
    //count_Rm[i]:左边界为i，右边界为m的区间数量
    vector<int> count_L1(m + 2, 0);
    vector<int> count_Rm(m + 2, 0);

    //左边界为1，右边界为m的区间数量
    int covered_all = 0;

    //一次遍历：统计
    for (int i = 1; i <= n; ++i) {
        int L, R;
        cin >> L >> R;
        //统计差分数组(策略一)
        diff[L]++;
        diff[R + 1]--;
        //统计左右两个部分(策略二)
        if (L == 1) count_L1[R]++;
        if (R == m) count_Rm[L]++;
        if (L == 1 && R == m) covered_all++;
    }

    //pref[k]:左边界为1，右边界 >= k 的区间数量
    //suff[k]:左边界 <= k，右边界为m的区间数量
    vector<int> pref(m + 2, 0), suff(m + 2, 0);

    for (int i = m; i >= 1; --i) {
        pref[i] = pref[i + 1] + count_L1[i];
    }
    for (int i = 1; i <= m; ++i) {
        suff[i] = suff[i - 1] + count_Rm[i];
    }

    int max_ans = 0;
    int cur = 0;
    //策略一：某个数字出现在区间最多的数量
    for (int i = 1; i <= m; ++i) {
        cur += diff[i];
        if (cur > max_ans) max_ans = cur;
    }

    //策略二
    for (int k = 1; k < m; ++k) {
        cur = pref[k] + suff[k + 1] - covered_all;
        if (cur > max_ans) max_ans = cur;
    }

    cout << max_ans << '\n';

    return 0;
}
```




## **常见易错点**
1. `m=1` 的边界情况：必须单独处理，否则会输出错误结果
2. 差分数组越界：`R+1` 最大为 `m+1`，因此数组大小必须至少为 `m+2`
3. 重复计算问题：全区间选手会被策略二的前后缀各算一次，必须减去 `covered_all`
4. 前后缀数组的计算方向：`pref` 数组必须从后往前算，`suff` 数组必须从前往后算
## **总结**
这道题的核心是利用选手区间的连续性，  
将一个看似指数级复杂度的划分问题，转化为线性时间可解的问题。  
关键在于理解 "集合等价于其最小包围区间" 这一引理，从而推导出最优划分只能是单点分割或前缀后缀分割两种情况。  
差分数组和前后缀预处理则是实现这一思路的高效技巧。
